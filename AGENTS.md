# ダッシュボード更新時の運用

- 工事情報や表示内容を更改した場合は、関連ファイル間の整合性を確認する。
- 公開情報に個人情報、連絡先、週報原文、内部資料などが含まれていないことを確認する。
- 更新後は表示・構文・Git差分を確認する。
- 週報の予定欄と実施欄を区別し、将来日は「予定」と明示する。日付が過ぎただけで予定を実績扱いせず、週報の実施欄を確認してから実績へ切り替える。
- 更新時は `data.js` の基準日、最新実績日、最終更新日時、確認資料数を合わせて見直す。
- 新しい表示項目や判断ルールを追加した場合は、別環境でも再現できるようREADMEへ理由と更新方法を追記する。
- 利用者からコミット・公開・プッシュの明示的な承認がある場合は、検証後に変更内容が分かる日本語のメッセージでコミットし、`origin` の作業ブランチへプッシュする。
- プッシュできない場合は、未完了であることと原因を利用者へ明示する。

## 使用材料機能を扱うとき

- 使用材料機能は2026年7月14日にローカル実装済み。更新前に `MATERIALS_IMPLEMENTATION_HANDOFF.md` を最後まで読む。
- 再利用ルールはCodexスキル `construction-progress-dashboard-handoff` の `references/material-catalogs.md` と `references/data-contract.md` を正とする。
- 材料原本フォルダは読み取り専用とし、変更・移動・改名しない。
- 資料を本文抽出する前に分類し、請求書、納品書、見積書、出荷案内書、領収書、注文書、価格表を要約・公開対象から除外する。
- 材料承認資料は製品選定の内部根拠にのみ使い、氏名、会社名、連絡先、押印、打合せ情報、私有パスを公開データへ転記しない。
- AI要約は補助情報とし、数値条件や安全事項を推測しない。公開時は最新版カタログ、製品表示、SDS、工事の承認条件を優先する注意書きを常時表示する。
- 公開用カタログとSDSは内容確認済みの複製またはメーカーの承認済みHTTPS URLだけを使う。Dropbox原本へ直接リンクしない。
- `materials` が未定義または空でも、工程表、カレンダー、実績、予定、写真が止まらず、材料専用ページは準備中表示になるよう実装する。
