# ダッシュボード更新時の運用

- 工事情報や表示内容を更改した場合は、関連ファイル間の整合性を確認する。
- 公開情報に個人情報、連絡先、週報原文、内部資料などが含まれていないことを確認する。
- 更新後は表示・構文・Git差分を確認する。
- 週報の予定欄と実施欄を区別し、将来日は「予定」と明示する。日付が過ぎただけで予定を実績扱いせず、週報の実施欄を確認してから実績へ切り替える。
- 更新時は `data.js` の基準日、最新実績日、最終更新日時、確認資料数を合わせて見直す。
- 新しい表示項目や判断ルールを追加した場合は、別環境でも再現できるようREADMEへ理由と更新方法を追記する。
- 検証に問題がなければ、変更内容が分かる日本語のメッセージでコミットし、`origin` の作業ブランチへプッシュするところまでを一連の作業として実行する。
- プッシュできない場合は、未完了であることと原因を利用者へ明示する。
